# Adaptive Dialog

## Adaptiveness

The adaptiveness behaviour changes based on the value of the `variant` prop

### Variant Tall

- Defaults to full screen mode on the extra-small breakpoint (cutomizable using the `fullScreenBreakpoint` prop)
- Stretches dialog actions on the extra-small breakpoint (cutomizable using the `stretchActionsBreakpoint` prop)

### Action Stretching

When actions are Stretched by the `stretchActionsBreakpoint` prop, they behave differently based on the number of actions:

- One or two: buttons are stacked horizontally and stretched to 50% of the dialog width
- More: buttons are stacked vertically and stretched to 100% of the dialog width

<Tabs>
  <Tab label="One Action">
    ```tsx
    import ExampleDialogContent from "@/shared/examples/exampleDialogContent";
    import Button from "@mui/material/Button";
    import AdaptiveDialog from "adaptive-material-ui/components/adaptiveDialog";
    import { useState } from "react";

    export default function Basic() {
      const [open, setOpen] = useState(false);

      return (
        <div>
          <Button onClick={() => setOpen(true)}>Open</Button>
          <AdaptiveDialog open={open} onClose={() => setOpen(false)} variant="short" stretchActionsBreakpoint={Number.MAX_SAFE_INTEGER}>
            <ExampleDialogContent numberOfActions={1} onClose={() => setOpen(false)} variant="short" />
          </AdaptiveDialog>
        </div>
      );
    }
    ```

  </Tab>
